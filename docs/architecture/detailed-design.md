# ShadowClaude 架构设计详细说明

## 1. 引言

ShadowClaude 采用现代化的分层架构设计，结合 Rust 的高性能和 Python 的灵活性，为 AI 编程助手领域提供了一个全新的解决方案。

### 1.1 设计原则

**性能优先原则**
Rust 核心确保关键路径的零成本抽象。所有热路径代码都使用 Rust 实现，包括查询处理、工具执行和内存管理。

**模块化设计原则**
每个功能模块都是独立的 crate 或 package，具有清晰的接口和单一职责。模块之间通过定义良好的 API 进行通信。

**渐进增强原则**
基础功能开箱即用，高级特性按需启用。用户可以根据需要选择启用 BUDDY、KAIROS、Undercover 等模块。

**安全第一原则**
六层权限纵深防御体系，确保 AI 助手在受控环境中执行操作。所有敏感操作都需要显式授权。

### 1.2 技术选型

**核心语言: Rust**
- 零成本抽象
- 内存安全保证
- 优秀的并发性能
- 跨平台支持

**扩展语言: Python**
- 丰富的生态
- 易于脚本编写
- 快速原型开发
- 广泛的开发者基础

**数据库:**
- Qdrant: 向量数据库用于语义记忆
- Neo4j: 图数据库用于情景记忆
- Redis: 内存缓存用于工作记忆

## 3. 详细设计

### 3.1 QueryEngine 设计

QueryEngine 是系统的核心入口，采用管道模式处理查询。

**处理流程:**
1. 输入解析
2. 意图识别
3. 上下文构建
4. 工具选择
5. 执行编排
6. 响应生成

**关键数据结构:**

```rust
pub struct Query {
    pub id: QueryId,
    pub content: String,
    pub context: Option<Context>,
    pub timestamp: DateTime<Utc>,
}

pub struct Response {
    pub id: ResponseId,
    pub content: String,
    pub tool_calls: Vec<ToolCall>,
    pub metadata: ResponseMetadata,
}
```

### 3.2 记忆系统设计

**三层记忆架构:**

语义记忆负责长期知识存储，使用向量数据库实现高效的相似度搜索。每个记忆条目包含：
- 内容文本
- 向量嵌入
- 元数据标签
- 时间戳

情景记忆记录具体事件和交互历史，使用图数据库存储事件之间的关系。支持：
- 时间序列查询
- 事件链追踪
- 模式识别

工作记忆维护当前会话的短期上下文，采用 LRU 淘汰策略。包括：
- 对话历史
- 活跃实体
- 临时状态

### 3.3 Agent 系统设计

基于 Actor 模型的 Agent 系统支持高并发和容错。

**Coordinator 职责:**
- Agent 注册和管理
- 任务分配和调度
- 结果收集和合并
- 错误处理和重试

**Swarm 模式:**
- 多 Agent 并行执行
- 任务自动分解
- 结果智能合并
- 负载动态均衡

## 4. 数据流

### 4.1 查询处理流

用户输入 → 预处理 → 意图识别 → 记忆检索 → Prompt构建 → LLM调用 → 工具执行 → 结果格式化 → 输出

### 4.2 记忆存储流

输入内容 → 类型分类 → 内容处理 → 嵌入生成 → 索引更新 → 持久化存储

### 4.3 Agent 协作流

任务提交 → 分解分析 → 依赖构建 → 并行调度 → 结果收集 → 质量检查 → 合并输出

## 5. 扩展性设计

### 5.1 插件系统

支持通过 Hook 机制扩展功能：

```rust
pub trait Hook {
    fn before_process(&self, query: &mut Query
    ) -> Result<()>;
    
    fn after_process(
        &self,
        query: &Query,
        response: &mut Response
    ) -> Result<()>;
}
```

### 5.2 MCP 集成

支持 Model Context Protocol，可连接外部工具服务器。
